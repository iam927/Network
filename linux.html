Linux Networking Commands

A reference guide to the most essential networking commands for Linux administration and troubleshooting.

1. ping

Tests reachability of a host on an Internet Protocol (IP) network and measures the round-trip time for messages.

ping -c 4 google.com
FlagDescription
-c [count]Stop after sending specified number of ECHO_REQUEST packets.
-i [interval]Wait interval seconds between sending each packet.

2. ip / ipconfig

The modern replacement for ifconfig. Shows or manipulates routing, network devices, interfaces, and tunnels.

ip a
ip route show
CommandDescription
ip addr showDisplay IP addresses and property information.
ip link set dev eth0 downBring an interface down.

3. netstat / ss

Print network connections, routing tables, interface statistics, masquerade connections, and multicast memberships. ss is the faster, modern alternative.

ss -tuln
FlagDescription
-tDisplay TCP sockets.
-uDisplay UDP sockets.
-lDisplay listening sockets.
-nDo not resolve service names (show ports instead).

4. traceroute

Prints the route packets trace to network host. It shows every hop along the path.

traceroute 8.8.8.8

5. nslookup / dig

Query Internet name servers interactively. dig provides more comprehensive output.

dig google.com
nslookup example.com
