The OSI Model

The Open Systems Interconnection (OSI) model conceptualizes how data travels across a network. Remembering the layers is critical for the CCNA exam.

Mnemonic Tip: Please Do Not Throw Sausage Pizza Away

1

Physical Layer

Deals with the physical transmission of raw data bits over a communication channel. (Cables, Hubs, Bits)

2

Data Link Layer

Provides node-to-node data transfer and error detection. Uses MAC addresses. (Switches, Frames)

3

Network Layer

Handles routing of data paths using logical IP addresses. (Routers, Packets)

4

Transport Layer

Ensures complete data transfer. Handles flow control and error recovery. (TCP/UDP, Segments)

5

Session Layer

Establishes, manages, and terminates connections between applications.

6

Presentation Layer

Translates, encrypts, and compresses data. Ensures data is readable by the receiving system.

7

Application Layer

Provides network services directly to the user's application. (HTTP, FTP, DNS)
